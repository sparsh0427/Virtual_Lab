Demonstration and Implementation of cryptographic algorithms.

Theory

Cryptography is the study and practice of techniques for secure communication in the presence of third parties called adversaries. It deals with developing and analyzing protocols which prevents malicious third parties from retrieving information being shared between two entities thereby following the various aspects of information security.

    Cryptography Components

  1. Plaintext and Ciphertext

    2. The original message, before being transformed, is called plaintext. After the message is transformed, it is called ciphertext. An encryption algorithm transforms the plaintext into ciphertext; a decryption algorithm transforms the ciphertext back into plaintext.





  2. Cipher

    3. In cryptography, a cipher (or cypher) is an algorithm for performing encryption or decryption—a series of well-defined steps that can be followed as a procedure.



  3. Key

    4. A key is a number (or a set of numbers) that the cipher, as an algorithm, operates on.



    Categories of Cryptography

  4. Symmetric key (also called secret-key) cryptography algorithms

    5. In symmetric-key cryptography, the same key is used by both parties. The sender uses this key and an encryption algorithm to encrypt data; the receiver uses the same key and the corresponding decryption algorithm to decrypt the data. In symmetric-key cryptography, the same key is used by the sender (for encryption) and the receiver (for decryption)
    Examples:
    XOR Cipher
    Data Encryption Standard (DES)
    Advanced Encryption Standard (AES)



  5. Asymmetric (also called public-key) cryptography algorithms

    6. In asymmetric or public-key cryptography, there are two keys: a private key and a public key. The private key is kept by the receiver. The public key is announced to the public. Examples:
    RSA
    Diffie-Hellman



    RSA ALGORITHM

    8. Anyone who needs to send a message to Bob can use n and e. For example, if Alice needs to send a message to Bob, she can change the message, usually a short one, to an integer. This is the plaintext. She then calculates the ciphertext, using e and n. C = P^e ( mod n ) Alice sends C, the ciphertext, to Bob.
